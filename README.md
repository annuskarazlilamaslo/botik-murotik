# 🐾 Ботик-муротик

Discord-бот для совместного прослушивания музыки. Каждый участник сервера может завести свою личную папку, загрузить туда любимые треки — и слушать их вместе с друзьями в голосовом канале, все вперемешку или выборочно.

Музыка хранится на **Яндекс.Диске**, бот подключается к нему через REST API и стримит треки прямо в голосовой канал Discord.

---

## ✨ Возможности

- 🎵 **Проигрывание музыки** — плейлист собирается из треков всех пользователей, воспроизведение в голосовом канале
- 📁 **Личные папки** — у каждого пользователя своя папка на Диске, привязанная к его Discord-аккаунту (по ID)
- 🎯 **Гибкая фильтрация** — можно слушать все папки сразу или выбрать, чьи треки играть, прямо из тех, кто сейчас в голосовом канале
- 🔁 **Режимы воспроизведения** — шаффл (включён по умолчанию) и зацикливание плейлиста
- 📋 **Лимит и учёт треков** — каждому доступно до 50 треков, с наглядным счётчиком при загрузке и просмотре (лимит можно изменить)
- 🔧 **Админ-панель** — управление чужими папками и треками через команды с префиксом `!админ_`
- 🐱 **Дружелюбный тон** — бот общается с пользователем на русском языке (также понимает и английские дубли команд)

## 🗂 Структура проекта

```
bot-music/
├── main.py                     # точка входа, регистрация основных команд
├── config.py                   # константы и чтение .env
├── texts.py                    # тексты помощи и прочие строковые константы
├── music_player.py             # класс MusicPlayer — плейлист и режимы воспроизведения

├── user_folders.json           # хранение привязок Discord ID → имя папки (создаётся автоматически или руками)
├── playback_selection.py       # временный фильтр папок для текущего прослушивания
├── admin.py                    # админские команды
├── user_folders.py             # пользовательские команды (папка, треки)
├── voice_folder_select.py      # интерактивный выбор папок из голосового канала

├── requirements.txt            # зависимости проекта
└── temp_tracks/                # временные аудиофайлы при воспроизведении (создаётся автоматически)
```

## ⚙️ Установка

### 1. Клонирование и окружение

```bash
git clone <ссылка на репозиторий>
cd bot-music
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Linux/macOS
pip install -r requirements.txt
```

### 2. FFmpeg

Бот использует FFmpeg для обработки и стриминга аудио — без него звук не заработает.

**Windows:**

1. Перейди на [gyan.dev/ffmpeg/builds](https://www.gyan.dev/ffmpeg/builds/)
2. Скачай **"release essentials"** (zip-архив) — этого достаточно, полная версия не нужна
3. Распакуй архив, найди внутри папку `bin`, а в ней файл `ffmpeg.exe`
4. Скопируй `ffmpeg.exe` в корень проекта (туда же, где `main.py`) — так бот найдёт его без лишних настроек

**Проверка, что всё скачалось правильно:**

```powershell
.\ffmpeg.exe -version
```

Если в ответ вывелась версия FFmpeg (а не ошибка "не найден файл") — всё в порядке, можно запускать бота.

**macOS / Linux**, если понадобится:

```bash
# macOS (через Homebrew)
brew install ffmpeg

# Ubuntu/Debian
sudo apt install ffmpeg
```

В этом случае FFmpeg установится в системный `PATH`, и путь к нему в `config.py` можно указать просто как `"ffmpeg"`, без полного пути к файлу.

### 3. Настройка `.env`

Создай файл `.env` в корне проекта и впиши туда переменные:

```
DISCORD_TOKEN=токен_твоего_бота
YANDEX_DISK_TOKEN=OAuth_токен_яндекс_диска
MUSIC_DIR = ./музыкальная папка/
FFMPEG_PATH = ./ffmpeg.exe
YANDEX_MUSIC_PATH = disk:/музыкальная_папка
ADMIN_IDS=discord_id_1,discord_id_2
```

**Discord Token** — создаётся в [Discord Developer Portal](https://discord.com/developers/applications). Обязательно включи и пригласи на сервер, дай боту права админа или пропиши их в настройках (на запись, чтение, подключение к голосовому каналу)
**Server Members Intent** и **Message Content Intent** в настройках бота (раздел Bot → Privileged Gateway Intents).

**Yandex Disk Token** — получается через [OAuth Яндекса](https://oauth.yandex.ru/), приложению нужно дать права:
- `cloud_api:disk.read` — чтение
- `cloud_api:disk.write` — запись

**Yandex Disk** - на самом Яндекс Диске нужно место для хранения треков. Заведи музыкальную папку, назови её, например, "music".

**Admin IDs** — Discord ID пользователей с доступом к админским командам. ID узнаётся через режим разработчика Discord (Настройки → Расширенные → Режим разработчика → правый клик по пользователю → Копировать ID). Если на канале планируется админский доступ нескольким админам, то ID нужно вписать через запятую без пробелов в .env, как указано в настройках к .env.

### 4. Запуск

```bash
python main.py
```

## 🎮 Команды

Полный список всегда можно посмотреть в самом Discord командой `!помощь` — там же появится дополнительный блок с админскими командами, если у тебя есть права.

### Музыка
| Команда | Действие |
|---|---|
| `!игра` / `!play` | начать проигрывание / снять с паузы |
| `!пауза` / `!pause` | поставить на паузу |
| `!дальше` / `!next`, `!skip` | следующий трек |
| `!прошлый` / `!prev`, `!back` | предыдущий трек |
| `!стоп` / `!stop`, `!leave` | остановить и выйти из канала |
| `!обновить` / `!update` | пересобрать плейлист с Диска |

### Режимы
| Команда | Действие |
|---|---|
| `!петля` / `!loop` | зациклить плейлист |
| `!смешать` / `!shuffle` | переключить обычный порядок / вразброс |

### Фильтр папок
| Команда | Действие |
|---|---|
| `!играть_папки` | выбрать, чьи папки играть, из присутствующих в голосовом канале |
| `!играть_все` | снять фильтр — играют все |

### Личная папка
| Команда | Действие |
|---|---|
| `!имя ТвоёИмя` | задать имя своей папки (один раз) |
| `!добавить` / `!загрузить`, `!add`, `!upload` | загрузить трек (прикрепить файл вместе с командой) |
| `!треки` | посмотреть список своих треков |

### Админ
| Команда | Действие |
|---|---|
| `!админ_имя @user Имя` | сменить привязку папки пользователя |
| `!админ_снести @user confirm` | удалить папку пользователя со всеми треками |
| `!админ_треки @user [файл]` | показать/удалить треки пользователя |
| `!админ_список` | показать все привязки папок |
